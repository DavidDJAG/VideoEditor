# Release checklist

## Toolchain detection
- Launch the app and run tool rescan.
- Verify ffmpeg, ffprobe, and ffplay paths are detected or clearly reported as missing.
- Verify startup diagnostics report codec and hardware acceleration status.

## Logs and diagnostics
- Run at least one successful and one failed queue job.
- Export diagnostics bundle and verify `jobs.json` plus per-job artifacts are present.
- Attach the diagnostics bundle path to release validation notes.

## Settings migration
- Start from a profile generated by the previous release.
- Upgrade to the new build and verify settings file can be read without data loss.
- Validate new settings fields fall back to defaults when absent.

## Packaging
- Build packaging artifact with `dotnet msbuild VideoEditor.UI/VideoEditor.UI.csproj /t:PackageInstaller /p:Configuration=Release`.
- Verify archive exists under `artifacts/installer`.
- Smoke-test extracted package on a clean Windows machine.
